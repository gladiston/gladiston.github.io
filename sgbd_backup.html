Estratégias de Backup

Diferenças entre backup completo, diferencial e incremental.

Introdução

Um DBA é tão bom quanto o seu último restore bem-sucedido. Entender as diferenças entre os tipos de backup permite otimizar o tempo de recuperação (RTO) e o ponto de recuperação (RPO).

Tipos de Backup

  • Completo (Full): Cópia total dos dados. Essencial, mas demorado e consome muito espaço.
  • Diferencial: Copia apenas o que mudou desde o último backup completo.
  • Incremental: Copia o que mudou desde o último backup de qualquer tipo. É o mais rápido, mas exige toda a cadeia para o restore.

O Caso do Firebird: NBackup

O Firebird utiliza o nbackup para realizar backups incrementais a nível de página física, permitindo cópias rápidas mesmo em bancos de centenas de gigabytes.

# Exemplo de comando para backup nível 0 (Full)
nbackup -user SYSDBA -password masterkey -backup 0 banco.fdb backup_full.nbk

Conclusão

A estratégia mais eficiente costuma ser um Full semanal com incrementais ou diferenciais diários, dependendo do volume de transações da empresa.
