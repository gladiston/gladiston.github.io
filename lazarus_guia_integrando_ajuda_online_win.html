Integrando o Editor com Ajuda On-line

Configuração manual de documentação CHM no Lazarus IDE.

  • Início
  • Guia de Sobrevivência

Quando usado o instalador oficial do Lazarus, o editor de código já vem integrado à ajuda on-line. Isso significa que, ao pressionar F1 com uma função ou método em foco, uma janela abre-se com a respectiva documentação. No entanto, em instalações via GIT ou fpcupdeluxe, essa integração precisa ser feita manualmente.

Localizando o Utilitário de Ajuda

Vá até a pasta:

$(LazarusDir)\components\chmhelp\lhelp

Lá você encontrará o executável lhelp (Linux/macOS) ou lhelp.exe (Windows). Se não o encontrar, certifique-se de que o pacote chmhelppkg.lpk foi compilado na IDE.

Baixando os Arquivos de Documentação

Agora, verifique a pasta:

$(LazarusDir)\docs\chm

Se ela estiver vazia, você precisará baixar os arquivos .chm oficiais. Acesse a página de documentação no SourceForge:

Configuração na IDE

Normalmente, a IDE detecta os arquivos automaticamente após a extração. Caso não funcione, acesse o menu Tools | Options | Help e selecione CHM Help Viewer:

  • HelpExe: Deve apontar para o lhelp.exe. Se vazio, a IDE busca o padrão em components\chmhelp\lhelp\.
  • HelpEXEParams: Geralmente vazio. Se usar o hh.exe do Windows (opcional), utilize "%s::%s".
  • HelpFilesPath: Onde estão os arquivos .chm. O padrão é $(LazarusDir)\docs\chm.

Conclusão

A documentação local (F1) é essencial para produtividade, especialmente ao explorar a LCL (Lazarus Component Library). Seguir estes passos garante que você tenha as definições de classes e métodos sempre à mão.
