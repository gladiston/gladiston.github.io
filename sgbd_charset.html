Qual charset devo usar?

Análise técnica entre ISO8859_1, WIN1252 e UTF8.

  • Início
  • Voltar para SGBD

Introdução

A escolha do Charset (conjunto de caracteres) é uma das decisões mais críticas na criação de um banco de dados. Uma escolha errada pode levar a problemas de acentuação, perda de dados em migrações e degradação de performance.

Comparativo de Formatos

  • ISO8859_1: Padrão clássico para línguas latinas. Ocupa 1 byte por caractere.
  • WIN1252: Uma variação da Microsoft do ISO, incluindo caracteres como o símbolo do Euro (€).
  • UTF8: O padrão moderno e universal. Suporta quase todos os caracteres do mundo, mas pode usar de 1 a 4 bytes por caractere.

Exemplo de Impacto

Em um banco Firebird, ao definir uma coluna como VARCHAR(100) CHARACTER SET UTF8, o motor reserva espaço para o pior cenário (400 bytes), o que afeta o cálculo do tamanho de página e índices.

Conclusão

Para sistemas novos, o UTF8 é a recomendação mandatória pela compatibilidade global. Para sistemas legados Windows que não sairão do Brasil, o WIN1252 ainda oferece a melhor economia de espaço e performance de indexação.
